haunted_field¶

File Haunted field¶

class src.problems.problem4.haunted_field.PlayerState[source]¶

This is used to define the possibles states of an individual through the haunted field An individual has five types of value:

  • active
  • success
  • monster
  • blocked
  • alive
class src.problems.problem4.haunted_field.HauntedField(height, width, nb_monsters)[source]¶
backup_field()[source]¶

Store a (deep) copy of self’s field, thus it is possible to apply modifications to self’s field and then to go back to initial field state using self.restore_field().

It can be used to store field before trying to “cross it” in order to restore it after crossing attempt.
